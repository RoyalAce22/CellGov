# spu_mailbox_write

Smallest possible PPU+SPU interaction. The SPU writes a known value
(0x1337BAAD) to the outbound mailbox, then stops.

## SPU binary

`spu_mailbox_write.bin` -- 16 bytes, 4 instructions, hand-assembled:

```
0x41099B83  ilhu  $3, 0x1337       # r3 = 0x13370000
0x60DD5683  iohl  $3, 0xBAAD       # r3 = 0x1337BAAD
0x20A00E03  wrch  $28, $3          # write r3 to SPU_WrOutMbox
0x00000000  stop                   # halt
```

## PPU wrapper ELF

`spu_mailbox_write.elf` -- 376 bytes, generated by `cellgov_mkelf`.

The PPU program (42 PPC64 instructions):

1. `sys_spu_initialize(1, NULL)` -- syscall 169
2. `sys_raw_spu_create(&id, NULL)` -- syscall 160
3. Copy SPU image to raw SPU 0 local store at `0xE0000000`
4. Set `SPU_NPC` to 0 via MMIO at `0xE0044034`
5. Write `SPU_RunCntl = 1` via MMIO at `0xE004401C`
6. Poll `SPU_MBox_Status` at `0xE0044014` until out_mbox count > 0
7. Read `SPU_Out_MBox` at `0xE0044004`
8. Store result to `0x20004`
9. `sys_process_exit(0)` -- syscall 22

Memory layout:

| Address    | Contents                        |
|------------|---------------------------------|
| `0x10000`  | OPD (function descriptor)       |
| `0x10008`  | PPU code (168 bytes)            |
| `0x20000`  | spu_id output (u32)             |
| `0x20004`  | mailbox result (u32)            |
| `0x20008`  | SPU image (16 bytes)            |

Regenerate: `cargo run -p cellgov_mkelf -- spu_mailbox_write tests/micro/spu_mailbox_write/spu_mailbox_write.elf`

## Expected behavior

- SPU writes 0x1337BAAD to channel 28 (SPU_WrOutMbox)
- SPU halts with stop type 0
- PPU reads 0x1337BAAD from the SPU's outbound mailbox
- PPU stores 0x1337BAAD at address 0x20004
